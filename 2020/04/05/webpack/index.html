# webpack 支持的模块化规范

  • commonjs 规范是给 webpack 配置文件使用的, AMD/ES6 等规范是给项目自己用的，所以会看到两种 js 规范的写法，不要奇怪😙
  • webpack 是基于 node 环境的，所以配置文件用 commonjs 规范
  • 配置文件就是 webpack 打包的依据

# webpack 局部安装和全局安装

  • 全局安装的 webpack 使你可以在任何地方使用 webpack 命令
  • 局部安装的 webpack 是在项目自身中使用。 使用的话需要指定路径
 ./node_modules/webpack/bin/webpack.js --config webpack.myconfig.js

webpack --config 文件名: 指定 webpack 打包时用的配置文件，默认是用的 webpack.config.js

  • 如果使用 script build 脚本，执行 npm run build 会优先使用项目自己的 webpack，没有则会使用全局

# 配置

const MiniCssExtractPlugin = require("mini-css-extract-plugin");

module.exports = {
  entry: {
    app: ["./app.js"],
  },
  output: {
    filename: "./[name].bundle.js", // hash: 作用截取, 其他内置变量[id][chunkhash]
  },
  plugins: [
    new MiniCssExtractPlugin({
      filename: "[name].min.css", // 输出文件名
      chunkFilename: "[id].css",
    }),
  ],
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [
          {
            loader: MiniCssExtractPlugin.loader,
            options: {
              // 这里可以指定一个 publicPath
              // 默认使用 webpackOptions.output中的publicPath
              publicPath: "../",
            },
          },
          "css-loader",
        ],
      },
    ],
  },
};

# webpack 环境变量(开发环境和生产环境)

# 理解

这里的开发环境和生产环境指的是项目业务层面，不同的环境有不同的业务参数，可以在打包时体现。

npm 的各种依赖，在哪个环境下使用？ 开发模式需要方便调试就不用压缩，treeshaking 等功能，要 webpack-dev-server。 生产环境要减少文件体积，就用压缩等功能。

development 打包出来的文件对开发友好 production 打包出来用于生产

这个和 npm 里的 dependence 和 devDependence 不一样。在 dependence 中是项目运行时用到的依赖, 比如一些业务上的库，在代码中引用的。 devDependence 是项目写代码时，打包时会用到的。和业务无关。

# 配置文件

不同环境的打包被分散到三个配置文件 webpack.common.js: 所有环境共通配置, 最后根据当前环境参数导出配置

const MiniCssExtractPlugin = require("mini-css-extract-plugin");
const { CleanWebpackPlugin } = require("clean-webpack-plugin");
const dev = require("./webpack.dev");
const prod = require("./webpack.prod");
const merge = require("webpack-merge");
module.exports = (env) => {
  const devMode = env !== "production";
  const common = {
    entry: "./src/app.js",
    output: {
      filename: "./bundle.js",
    },
    module: {
      rules: [
        {
          test: /\.css$/,
          use: [
            {
              loader: MiniCssExtractPlugin.loader,
              options: {
                hmr: process.env.NODE_ENV === "development",
              },
            },
            "css-loader",
          ],
        },
      ],
    },
    plugins: [
      new MiniCssExtractPlugin({
        filename: devMode ? "app.dev.css" : "app.bundle.css",
        chunkFilename: devMode ? "[id].css" : "[id].[hash].css",
      }),
      new CleanWebpackPlugin(),
    ],
  };
  return merge(common, env === "production" ? prod : dev);
};

# 代码分割

  • 多页面: 共通模块分割出来单独打包 单独为独立文件后浏览器可以进行缓存，多个页面引用时就不用重复加载 主业务代码+公共依赖+第三方包+webpack 运行代码
  • 单页面: 减少单文件体积，把可以异步加载的模块改成异步加载 主业务代码+一步模块+第三方包+webpack 运行代码
  • 业务代码纯净，拆分后会形成三个文件 app.js // 业务代码 vender.js // 第三方模块 manifest.js // webpack 运行代码
